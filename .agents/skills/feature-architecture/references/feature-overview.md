# Feature Overview

## Rules
- Продуктовые фичи создаются с применением API/Impl split
- Все фичи лежат в features папке
- Один feature модуль может содержать один экран или несколько экранов (продуктовый флоу) или ui компонет, не являющийся экранов, но имеющим определенную свою логику, или фича без ui

## Naming

- имя модуля пишется с применением camelCase

## Верхнеуровневые правила для фич без UI

- Фичи без ui в api модуле содержат только модели и api классы, которые будут реализованы в impl модуле

## Верхнеуровневые правила для флоу экранов

- В апи модуле должна содержаться только одна FeatureFactory, которая представляет собой рутовый экран флоу в фиче (компонент, содержащий в себе другие)
- Всегда у флоу фичей должен быть отдельный root компонент, даже если ничего принципиально полезного он не делает
- Другие экраны содержат в navigation папке свои internal FeatureFactory, но уже без деления на интерфейс и реализацию, сразу реализация

## Верхнеуровневые правила для фичи с одним экраном или одним ui компонентом

- В апи модуле должна содержаться только одна FeatureFactory, которая представляет основную вью фичи=
